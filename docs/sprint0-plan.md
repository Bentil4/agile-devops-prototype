# Sprint 0 - Planning Sprint

## Sprint Goal
Establish the project foundation, define the product vision, create a prioritized backlog, and prepare for execution sprints.

## Duration
2 days

## Deliverables Completed

### 1. Product Vision
A lightweight REST API that enables users to create, manage, and track tasks efficiently, helping individuals organize their daily work and boost productivity.

### 2. Technology Stack Selected
- **Backend:** Node.js 18+ with Express
- **Testing:** Jest with coverage
- **CI/CD:** GitHub Actions
- **Version Control:** Git + GitHub

### 3. Product Backlog Created
- 7 user stories defined
- Acceptance criteria added to all stories
- Story points estimated using Fibonacci scale
- Stories prioritized (High, Medium, Low)

### 4. Definition of Done Established
Created comprehensive DoD covering code quality, testing, CI/CD, and documentation standards.

### 5. Sprint 1 Planning
Selected stories for Sprint 1:
- Story 7: Health Check Endpoint (1 point)
- Story 1: Create a Task (3 points)
- Story 2: View All Tasks (2 points)
- Story 5: Mark Task as Complete (2 points)

**Total Sprint 1 Capacity:** 8 story points

## Key Decisions Made

1. **In-Memory Storage:** Using simple in-memory storage for the prototype
2. **REST API Focus:** Building API-only (no frontend)
3. **Minimal Dependencies:** Using only essential libraries
4. **GitHub Actions:** Selected for CI/CD
